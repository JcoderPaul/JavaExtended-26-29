******* Enum StandardOpenOption *******

public enum StandardOpenOption extends Enum<StandardOpenOption>
                               implements OpenOption

Определяет стандартные параметры открытия.

- APPEND - Если файл открыт для записи, то байты будут записаны в конец файла, а не в начало.
           Если файл открыт для записи другими программами, то это зависит от файловой системы,
           если запись в конец файла является атомарной.

- CREATE - Создает новый файл, если он не существует. Эта опция игнорируется, если уже установлена
           опция CREATE_NEW. Проверка существования файла и создание файла, если он не существует,
           атомарны по отношению к другим операциям файловой системы.

- CREATE_NEW - Создает новый файл, если файл уже существует. Проверка существования файла и создание
               файла, если он не существует, атомарны по отношению к другим операциям файловой системы.

- DELETE_ON_CLOSE - Удаляет при закрытии. Когда эта опция присутствует, реализация делает все возможное,
                    чтобы удалить файл, когда он закрыт соответствующим методом закрытия. Если метод
                    close не вызывается, то предпринимаются все усилия, чтобы удалить файл, когда
                    виртуальная машина Java завершает работу (либо обычно, как определено в спецификации
                    языка Java, либо, где возможно, аварийно). Этот параметр в первую очередь предназначен
                    для использования с рабочими файлами, которые используются исключительно одним
                    экземпляром виртуальной машины Java. Этот параметр не рекомендуется использовать при
                    открытии файлов, которые одновременно открыты другими объектами. Многие детали
                    относительно того, когда и как файл удаляется, зависят от реализации и поэтому не
                    указаны. В частности, реализация может быть не в состоянии гарантировать удаление
                    ожидаемого файла при его замене злоумышленником, пока файл открыт. Следовательно,
                    чувствительные к безопасности приложения должны соблюдать осторожность при
                    использовании этой опции.

                    Из соображений безопасности этот параметр может подразумевать параметр
                    LinkOption.NOFOLLOW_LINKS. Другими словами, если параметр присутствует при открытии
                    существующего файла, который является символической ссылкой, он может завершиться
                    ошибкой (выбрасывая исключение IOException).

- DSYNC - Требует, чтобы каждое обновление содержимого файла записывалось синхронно на базовое
          устройство хранения.

- READ - Открыть для чтения.

- SPARSE - Разреженный файл. При использовании с параметром CREATE_NEW этот параметр указывает на то,
           что новый файл будет разреженным. Параметр игнорируется, если файловая система не поддерживает
           создание разреженных файлов.

- SYNC - Требуется, чтобы каждое обновление содержимого или метаданных файла записывалось
         синхронно на базовое устройство хранения.

- TRUNCATE_EXISTING - Если файл уже существует и открыт для записи, то его длина усекается до 0.
                      Эта опция игнорируется, если файл открывается только для чтения.

- WRITE - Открыть для записи.
